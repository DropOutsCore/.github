## Who We Are

DropOutsCore is a focused team of builders — developers and designers who treat every deadline like a launchpad and every idea like a hypothesis worth pressure-testing. We started as a hackathon crew, stayed because the problems got interesting, and now we ship AI tools, full-stack products, and experimental builds that sit somewhere between *genuinely useful* and *let's see if this holds in prod*. We operate with startup velocity, an open-source mindset, and the shared conviction that the best way to understand a problem is to ship a solution to it — preferably before the sun comes up.

Built on curiosity, caffeine, and questionable decisions.

---

## 🔨 What We Build

```
AI / ML          →  Predictive models, LLM-powered tools, inference APIs that don't embarrass us
Web Applications →  Full-stack products with real users and real uptime requirements
Hackathon Builds →  36-hour sprints. Questionable architecture. Surprisingly functional demos.
Experimental     →  The kind of tool that has no business working — but does.
```

We pick problems that feel slightly too ambitious. That's the point.

---

## 🧠 Our Philosophy

**Ship it, then sharpen it.** A working prototype at 3am beats a perfect spec that never runs. We move fast, iterate in public, and let the code speak louder than the pitch.

**Depth over demo.** We read the papers. We write the tests — eventually. We care about what's under the hood, not just what renders in the browser tab the judge opens.

**Build together, not alone.** Every repo here has been touched by more than one pair of hands. Good code is reviewed code. Great ideas come from disagreements in a group chat at midnight.

> *DropOuts by name. Proven by code.*

## 🤝 Join the Chaos

We're selectively open. If you:

- Commit before you're fully confident and debug from the error messages
- Have refreshed a CI tab more times than you've refreshed a social feed
- Think "it works on my machine" is a valid first step toward "it works everywhere"
- Build things before asking for permission to build them

...there's probably a repo here with your name on it.

| Action | What it actually means |
|--------|------------------------|
| ⭐ Star a repo | We notice. It fuels the next sprint. |
| 🐛 Open an issue | We'll read it. Probably fix it. |
| 🔀 Submit a PR | Bold move. Respect. |
| 📬 Reach out | Links below. Don't overthink it. |

We're not hiring. We're collaborating. There's a difference.
